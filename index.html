JSON Lines

Documentation for the JSON Lines text file format

Simple. Like JSON

This page describes the JSON Lines text format. JSON Lines is a convenient format for storing structured data that may be processed one record at a time. It works well with unix-style text processing tools and shell pipelines.

JSON Lines files may be saved with the file extension .jl.

Stream compressors like gzip or bzip2 are recommended for saving space, resulting in .jl.gz or .jl.bz2 files.

UTF-8 Encoding

UTF-8 is backwards compatible with ASCII. JSON allows encoding Unicode strings with only ASCII escape sequences, however those escapes are ugly in a text editor.

Choosing UTF-8 as the standard lets the JSON Lines author choose to escape characters or not. Also, if a file is saved with a different encoding it is very unlikely to be valid UTF-8, making the mistake quickly obvious.

Each Line is a Valid JSON Value

The most common values will be objects or arrays, but any of the following is valid:

  • string
  • number
  • object
  • array
  • true
  • false
  • null

See json.org for more information.

Line Separator is '\n'

This means '\r\n' is also supported because white space within lines is ignored, as dictated by the JSON spec.

The last character in the file may be a line separator, and it will be treated the same as if there was no line separator.

Count Lines From 1

Text editing programs call the first line of a text file "line 1". We also call the first JSON value in a JSON Lines file value 1.
