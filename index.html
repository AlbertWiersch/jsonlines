JSON Lines

Documentation for the JSON Lines text file format

Simple. Like JSON

This page describes the JSON Lines text format. JSON Lines is a convenient format for storing structured data that may be processed one record at a time. It works well with unix-style text processing tools and shell pipelines.

JSON Lines files may be saved with the file extension .jl.

Stream compressors like gzip or bzip2 are recommended for saving space, resulting in .jl.gz or .jl.bz2 files.

UTF-8 Encoding

UTF-8 is backwards compatible with ASCII. JSON allows encoding Unicode strings with only ASCII escape sequences, however those escapes may be considered ugly when viewed in a text editor.

Choosing UTF-8 as the standard lets the JSON Lines author choose to escape characters or not. Also, if a file is saved with a different encoding it is very unlikely to be valid UTF-8, making the mistake easy to identify.

Each Line is a Valid JSON Value

The most common values will be objects or arrays, but any JSON value is permitted.

See json.org for more information about JSON values.

Line Separator is '\n'

This means '\r\n' is also supported because trailing white space is ignored when parsing JSON values.

The last character in the file may be a line separator, and it will be treated the same as if there was no line separator present.

Count Lines From 1

Text editing programs call the first line of a text file "line 1". We also call the first value in a JSON Lines file "value 1".

Examples

Better than CSV

["Name", "Session", "Score"]
["Gilbert", "2013", 24]
["Alexa", "2013", 29]
["May", "2012B", 14]
["Deloise", "2012A", 19]

Easy Nested Data

{"name": "Gilbert", "wins": [["straight", "7♣"], ["one pair", "10♥"], ["high card", "K♦"]]}
{"name": "Alexa", "wins": [["two pair", "4♠"], ["two pair", "9♠"]]}
{"name": "May", "wins": []}
{"name": "Deloise", "wins": [["three of a kind", "5♣"]]}
